user-invocable: false
---

# Use this skill when:
- defining scoring logic
- designing ranking strategy
- defining tie-breaking
- producing a minimal architecture

# Implementation Requirements:
- keep the design minimal
- prefer deterministic logic
- avoid unnecessary architecture
- include a simple data flow
- include a small architecture diagram
- do not introduce external systems

# Output Format:
- technical approach
- scoring rules
- ranking and tie-breaking logic
- function responsibilities
- architecture diagram (Mermaid)

# What this is NOT:
- not feature definition
- not implementation code
- not code review
- not system redesign
