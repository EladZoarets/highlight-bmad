# Use this skill when:
- interpreting the assignment
- defining feature behavior
- identifying inputs and outputs
- defining success criteria

# Implementation Requirements:
- focus only on the requested feature
- keep requirements concise and testable
- distinguish required behavior from optional ideas
- align strictly with the assignment

# Output Format:
- feature summary
- input definition
- output definition
- success criteria
- constraints

# What this is NOT:
- not a technical design
- not an implementation plan
- not a code generation step
- not architecture definition
