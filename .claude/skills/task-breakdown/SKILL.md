user-invocable: false
---

# Use this skill when:
- converting a technical plan into executable tasks
- preparing implementation order
- reducing ambiguity before coding
- ensuring work can be done in small steps

# Implementation Requirements:
- tasks must be small and concrete
- tasks must be executable in sequence
- each task should have a clear outcome
- avoid combining multiple major changes in one task
- align tasks with the approved architecture

# Output Format:
- ordered task list
- one-line purpose per task
- expected output per task

# What this is NOT:
- not feature definition
- not technical architecture
- not implementation code
- not validation
